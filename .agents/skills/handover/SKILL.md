# handover skill

## スキル概要

セッション中にいつでも実行できる引き継ぎドキュメント生成スキル。
現在のセッションを振り返り、次のセッションで迷わないようにするための `MEMORY.md` をVault直下に作成する。

## 役割

シフト交代レポートのように、次のClaudeセッションに現在の状況を正確に伝えること。

## 出力先

`Obsidian/MEMORY.md`（Vault直下）

## MEMORY.mdの構成

以下のセクションを含むMarkdownファイルを生成すること。

```markdown
# MEMORY - <YYYY-MM-DD HH:MM>

### 作業内容のサマリー

このセッションで何をしていたか、何が完了したかを簡潔にまとめる。

### 完了したこと

- 完了したタスクをリストアップする

### うまくいかなかったこと・バグと対処

- 発生した問題、バグ、エラーと、どう解決したか（または未解決か）を記載する

### 重要な意思決定とその理由

- セッション中に行った判断・選択とその根拠を記載する

### 学んだこと・注意点

- 気づき、ハマりポイント、次回への注意事項を記載する

### 次のステップ

- 次のセッションで続けるべきタスクや確認事項をリストアップする

### 重要なファイルマップ

| ファイルパス | 役割・説明 |
| --- | --- |
| path/to/file.md | ファイルの役割 |
```

## 過去ファイルの取り扱い

過去のファイルは`.agents/memories/handover`直下に移動させること。

この時、ファイル名を`memory_yyyymmddhhmmss.md`に変更すること。

## 生成ルール

- セッションの会話履歴全体を参照して内容を生成すること
- 日本語で出力すること
- 各セクションは具体的かつ簡潔に記載すること（箇条書き推奨）
- 「次のステップ」は次のセッションの人間がすぐ行動できる粒度で書くこと
- ファイルマップはセッション中に触れた・作成した・重要なファイルのみ記載すること
- テーブルはパイプ（`|`）の前後に必ずスペースを入れること
